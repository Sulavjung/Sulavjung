# Sulav
> [!IMPORTANT]
> Intellectual growth is a core priority for me. I look for environments that challenge thinking, encourage learning, and support long-term development. If that is not something your organization values, I would not be a good fit.

I don’t just use technology. I try to understand how it works and rebuild it better. I spend a lot of time thinking about problems, breaking them down, and finding efficient solutions. Inefficiencies in systems stand out to me and I feel the need to fix them.

I recently completed my bachelor’s in computer science, where I worked across full stack development, backend systems, and low level programming. I have built real systems used in small businesses, worked on startup ideas, and explored everything from web applications to operating systems and networking.

Right now, I work as an L2 Autonomous Vehicle Operator, where I am directly involved in collecting data for autonomous systems. This has pushed my interest toward the other side of the problem, which is working with the data itself using machine learning and deep learning.

Lately, I have been focused on studying machine learning. I read consistently and usually carry two books with me. One constant is Algorithms in a Nutshell, and I always keep another book based on what I am learning at the moment, currently Deep Learning with PyTorch.

My goal is to move deeper into machine learning, autonomous systems, and research driven work over the next few years.

## B. Work and Projects

I have worked across different types of systems, from user facing applications to low level programming.

1. On the application side, I co founded a project where we built an AI driven task management and journaling system using React, Express, MySQL, and Docker, deployed on AWS. This project evolved from a class project into something closer to a startup and required thinking about scalability, architecture, and user experience.

2. In retail, I built tools that are used daily in a liquor store, including a label creator and inventory related systems. These tools improved efficiency and reduced manual work by integrating barcode scanning and structured data input.

3. I have also built several systems level projects during my studies. These include a custom HTTP server in Java, a Linux shell in C, a file system, and a device driver. These projects helped me understand how systems actually work at a lower level, including memory, processes, and communication with hardware.

4. Other projects include a real time multiplayer poker game using WebSocket and PostgreSQL, a personal finance tracker built with TypeScript, and a full stack video sharing platform.

Across all of these, the consistent goal has been to build systems that are efficient, practical, and grounded in real use cases.
